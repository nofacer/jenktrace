# jenktrace v1.0.8

jenktrace 1.0.8 adds a build-status filter block above the build list so operators can narrow recent builds to the statuses they care about without changing the active analytics range.

## Highlights

- Added a new filter block above the build list in the job details view.
- Added status filter options for the statuses present in the current build data, plus an All option.
- Kept build analytics charts and summary metrics scoped only by the selected time range while the new filter affects only the build list.

## Included in this release

- Added shared build-status filter helpers for Jenkins build records.
- Added tests covering build-status classification, filtering, and option generation.
- Bumped packaged app version metadata to 1.0.8.

## Notes

- Filter options adapt to the statuses present in the current build list.
- If the selected status disappears after a refresh or time-range change, the filter resets to All.
