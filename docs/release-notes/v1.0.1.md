# jenktrace v1.0.1

jenktrace 1.0.1 improves Jenkins HTTPS connectivity for environments that rely on internally managed or enterprise-trusted certificate authorities.

## Highlights

- Added an instance-level option to disable SSL verification for Jenkins connections that use self-signed or otherwise untrusted certificates.
- Improved macOS app startup so packaged builds can launch Bun with `--use-system-ca`, allowing Jenkins HTTPS requests to use the system trust store.
- Updated connection testing and Jenkins data fetch flows to respect the new SSL behavior consistently across instance validation, job details, monitoring, and build log retrieval.

## Included in this release

- New Jenkins instance setting: `Disable SSL verification`.
- macOS packaged app hook to inject `BUN_OPTIONS=--use-system-ca` into the app bundle environment.
- Internal cleanup to rely on process-level system CA behavior instead of duplicating system certificate loading in request code.

## Notes

- The macOS system CA behavior applies to newly packaged app bundles that include the updated `Info.plist` environment entry.
- `Disable SSL verification` remains available as a per-instance fallback when a Jenkins endpoint still fails certificate validation.
