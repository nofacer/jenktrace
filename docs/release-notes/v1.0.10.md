# jenktrace v1.0.10

jenktrace 1.0.10 keeps the Jenkins details panel visible during manual and automatic refreshes so operators no longer see the right side blank out while updated data is loading.

## Highlights

- Kept the existing job details content mounted during refreshes when prior data is already available.
- Preserved the current analytics cards, charts, and build list while refresh requests are in flight.
- Limited loading skeletons to the initial empty-state load before any job data has been fetched.

## Included in this release

- Stopped clearing cached job details, activity, and analytics state when refresh requests fail.
- Added a shared details-panel load-state helper and tests covering refresh-versus-initial-load behavior.
- Bumped packaged app version metadata to 1.0.10.

## Notes

- Selecting a different instance or job still clears the prior job state before loading the newly selected target.
- Error alerts continue to render during refreshes without hiding the previously loaded content.
