# jenktrace v1.0.3

jenktrace 1.0.3 fixes a macOS packaging issue where the first launch could replace the outer app wrapper with the extracted inner app bundle and lose the `BUN_OPTIONS=--use-system-ca` environment setting.

## Highlights

- Fixed macOS first-launch behavior so the extracted app bundle preserves the `--use-system-ca` Bun startup option after self-extraction completes.
- Ensured Jenkins HTTPS requests continue to use the macOS system trust store after the app relaunches from the installed inner bundle.
- Reduced the risk of certificate validation regressions that only appeared after the first packaged app launch.

## Included in this release

- Added a macOS build-time plist injection step for the inner packaged `.app` bundle.
- Kept the existing wrapper-level plist injection so both the self-extracting launcher and the installed app bundle carry the same `BUN_OPTIONS` value.
- Updated project documentation to describe jenktrace as an end-user desktop app instead of a development scaffold.

## Notes

- This fix is relevant to newly packaged macOS builds of 1.0.3 and later.
- Existing per-instance SSL fallback behavior remains unchanged.
