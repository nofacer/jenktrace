# jenktrace v1.0.4

jenktrace 1.0.4 keeps the selected job details page fresh automatically so operators do not need to manually refresh to see current Jenkins state.

## Highlights

- Added automatic refresh for the selected job details, activity, and analytics views every minute while the page is visible.
- Prevented overlapping automatic refresh requests so the details view does not start duplicate loads while a prior refresh is still in flight.
- Skipped background page refreshes when the app window is not visible, reducing unnecessary Jenkins traffic.

## Included in this release

- Refactored the main view job-loading logic into shared refresh callbacks for initial load, manual refresh, and timed refresh.
- Kept the existing manual refresh action available for explicit operator-driven reloads.
- Bumped the packaged app version metadata to 1.0.4.

## Notes

- Automatic page refresh currently runs on a fixed 60-second interval.
- Background Jenkins monitoring remains separate from the foreground details page refresh behavior.
