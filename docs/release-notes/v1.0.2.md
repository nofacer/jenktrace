# jenktrace v1.0.2

jenktrace 1.0.2 improves build history responsiveness when switching time ranges and reduces intermittent RPC timeout failures in the job details view.

## Highlights

- Fixed an issue where changing the build history time range could trigger a full Jenkins job refresh instead of only reloading local analytics data.
- Increased the frontend RPC request timeout from 1 second to 10 seconds so slower Jenkins responses are less likely to fail prematurely.
- Improved loading behavior in the job details screen so analytics refreshes show scoped loading states without blanking the entire details view.

## Included in this release

- Separated job details and job analytics refresh flows in the main view.
- Added request sequencing guards to prevent stale RPC responses from overwriting newer state.
- Updated analytics and build list loading placeholders to reflect time-range refreshes more clearly.

## Notes

- Time-range changes now rely on the locally persisted analytics query path and should no longer trigger unnecessary Jenkins network fetches.
- Manual job refresh still reloads Jenkins-backed details and activity, while analytics continue to respect the currently selected time range.
