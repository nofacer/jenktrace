# jenktrace v1.0.6

jenktrace 1.0.6 adds an application log panel so operators can inspect what the desktop app itself is doing, including recent Jenkins HTTP request outcomes.

## Highlights

- Added a new app log panel in the main details area with recent runtime events, refresh controls, and repeated-event counts.
- Captured app-level events for startup, background monitoring ticks, Jenkins instance changes, monitored job changes, and cached build logs.
- Logged Jenkins HTTP request outcomes with sanitized URLs, response statuses, durations, and network failures for easier troubleshooting.

## Included in this release

- Added a shared app log model and SQLite-backed runtime log store for persisted operational logs.
- Routed Jenkins API requests and connection tests through the new logging helper so request status and timing are recorded consistently.
- Bumped packaged app version metadata to 1.0.6.

## Notes

- App logs are refreshed automatically while the window is visible.
- HTTP log entries intentionally omit query strings and fragments from recorded URLs.
